Команди git

Загальне налаштування

  • $ git config --global user.name "[ім'я]" - Встановлює ім'я, яке буде відображатися в полі автора у комітів, які ви виконуєте.
  • $ git config --global user.email "[адреса електронної пошти]" - Встановлює адресу електронної пошти, яка відображатиметься в інформації про коміти, які ви виконуєте.

Внесення змін

  • $ git status - Перераховує всі нові або змінені файли, які потребують фіксації
  • $ git diff - Показує відмінності щодо внесених змін у ще не проіндексованих файлах
  • $ git add [файл] - Індексує вказаний файл для наступного коміту
  • $ git diff --staged - Показує різницю між проіндексованою та останньою зафіксованою версіями файлів
  • $ git reset [файл] - Скасовує індексацію вказаного файлу, зберігаючи його вміст.
  • $ git commit -m "[повідомлення з описом] - Фіксує проіндексовані зміни та зберігає їх в історію версій

Створення репозиторію

  • $ git init [назва проекта] - Створює новий локальний репозиторій із заданим ім'ям
  • $ git clone [url-адрес] - Завантажує репозиторій разом із усією його історією змін

Колективна робота

  • $ git branch - Список іменованих гілок коммітів із зазначенням обраної гілки
  • $ git branch [ім'я гілки] - Створює нову гілку
  • $ git switch -c [ім'я гілки] - Перемикається на вибрану гілку та оновлює робочу директорію до її стану
  • $ git merge [ім'я гілки] - Вносить зміни вказаної гілки в поточну гілку
  • $ git branch -d [ім'я гілки] - Видаляє вибрану гілку

Операції з файлами

  • $ git rm [файл] - Видаляє конкретний файл із робочої директорії та індексує його видалення
  • $ git rm --cached [файл] - Забирає конкретний файл із контролю версій, але фізично залишає його на своєму місці
  • $ git mv [оригінальный файл] [нове ім'я] - Переміщує та перейменовує вказаний файл, одразу індексуючи його для наступного комміту

Перегляд історії

  • $ git log - Історія коммітів для поточної гілки
  • $ git log --follow [файл] - Історія змін конкретного файлу, включаючи його перейменування
  • $ git diff [первая ветка]...[вторая ветка] - Показує різницю між змістом коммітів двох гілок
  • $ git show [коммит] - Виводить інформацію та показує зміни у вибраному коментарі

Відкот комітів

  • $ git reset [коммит] - Скасує всі коміти після заданого, залишаючи всі зміни у робочій директорії
  • $ git reset --hard [коммит] - Скидає всю історію разом із станом робочої директорії до вказаного комміту.

Синхронізація з віддаленим репозиторієм

  • $ git fetch [удалённый репозиторий] - Завантажує всю історію з віддаленого репозиторію
  • $ git merge [удалённый репозиторий]/[ветка] - Вносить зміни з гілки віддаленого репозиторію в поточну гілку локального репозиторію
  • $ git push [удалённый репозиторий] [ветка] - Завантажує всі зміни локальної гілки у віддалений репозиторій
  • $ git pull - Завантажує історію з віддаленого репозиторію та поєднує її з локальною. pull = fetch + merge

Збереження фрагментів

  • $ git stash - Тимчасово зберігає всі незафіксовані зміни файлів, що відстежуються.
  • $ git stash pop - Відновлює стан раніше збережених версій файлів
  • $ git stash list - Виводить список усіх тимчасових збережень
  • $ git stash drop - Скидає останні тимчасово збережені зміни
